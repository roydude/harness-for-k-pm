# Skill: korean-humanizer

## 언제 쓰는가

- 사용자가 "휴머나이징", "번역체 제거", "AI 티 없애줘", "사람처럼 말해줘" 등을 요청할 때
- 한국어 텍스트가 게시 목적(블로그, 보고서, SNS, 보도자료)인데, 현재 AI스럽거나 번역투이거나 템플릿처럼 읽힐 때
- 한국어 텍스트의 가독성과 자연스러움을 높여야 할 때

## 필수 입력

| 항목 | 설명 |
|---|---|
| **원문 텍스트** | 다듬을 대상 텍스트 (+ 서식 제약이 있으면 함께) |
| **장르** | blog / internal report / SNS / press release — 모르면 맥락에서 추론하고 가정을 밝힌다 |
| **톤·존칭 레벨** | `합니다체` / `해요체` / `해라체` — 기본값: 원문의 지배적 스타일 유지 |
| **하드 제약** | 유지할 사실, 용어, 길이 제한, 반드시 살려야 할 문구 (있을 경우) |

## 워크플로우

1. **톤 고정** — 존칭 레벨 하나를 정하고 전체에서 일관되게 유지한다.
2. **1차 패스 (K-패턴)** — 한국어 구조 문제부터 잡는다 (조사 '의' 남용, 명사화, 피동, 번역투 등).
3. **2차 패스 (C-패턴)** — 범용 AI 글쓰기 신호를 제거한다 (과장, 모호한 출처, 접속부사 과잉, 군더더기 등).
4. **리듬·목소리** — 문장 길이에 변화를 주고, 장르에 맞으면 1인칭·구어 요소를 가볍게 넣는다.
5. **의미 점검** — 최종 읽기를 통해 의미, 강조점, 하드 제약이 그대로인지 확인한다.

## 빠른 점검 체크리스트

- '의'가 연쇄로 붙어 딱딱함 → 문장 쪼개기 / 동사화
- "~할 수 있습니다 / 것입니다 / …으로 보입니다" 반복 → 단정 / 구체화 / 불확실성 명시
- "다양한 / 효과적인 / 혁신적인 / 핵심적인" 같은 범용 형용사 → 구체적 사실·예시로 대체 (새 사실은 만들지 않는다)
- 접속부사(또한 / 더불어 / 나아가) 연속 → 삭제 / 문장 합치기
- 명사화·피동 남용 → 능동 + 동사 중심으로 재구성
- 문단 리듬이 전부 같음 → 짧게·길게 섞어 리듬 만들기

## 출력 형식

1. **다듬은 텍스트** — 바로 붙여넣기 가능한 상태
2. *(선택, 간략하게)* **변경 요약** — 어떤 패턴을 어디서 고쳤는지 3~7개 불릿

## 제약 사항 (반드시 지킨다)

- 새로운 사실, 숫자, 인용, 출처를 **만들어내지 않는다**. 근거가 필요하면 요청하거나, 주장을 적절히 모호하게 둔다.
- 저자의 의도를 보존한다. 장르가 보도자료가 아닌 한 "보도자료 톤"으로 과하게 다듬지 않는다.
- 문서의 서식 구조를 유지한다 (구조 변경 요청이 없는 한).

## 레퍼런스

전체 패턴 카탈로그(K1~K8, C1~C10)와 장르별 톤 가이드는 `references/patterns.md`를 참고한다.
