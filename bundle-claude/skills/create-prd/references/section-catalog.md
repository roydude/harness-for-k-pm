# PRD 섹션 카탈로그

기본 구조(기획 개요/목적·목표/검증 가설/핵심 경험/기술 구조/시나리오/데이터/의사결정/부록) 외에
PM이 추가할 수 있는 섹션 목록이다.
Phase 5에서 PM에게 이 목록을 제안하고, 원하는 섹션을 선택하게 한다.

> 주의: "사용자 시나리오"와 "데이터 구조"는 기본 템플릿에 이미 포함되어 있으므로
> 여기에는 기본 템플릿에 없는 추가 섹션만 나열한다.

---

## 선택 가능한 추가 섹션

### Out of Scope (범위 밖 정의)
이번 버전에서 의도적으로 포함하지 않는 것을 명시한다.
"이건 왜 안 들어가요?"라는 질문을 사전에 차단하는 역할.

```markdown
## Out of Scope
- [제외 항목 1]: [제외 이유]
- [제외 항목 2]: [제외 이유]
```

### 기술 요구사항 (Technical Requirements)
개발팀이 알아야 할 기술적 제약이나 요구사항.

```markdown
## 기술 요구사항
### API 명세
- Endpoint: `[METHOD] /api/v1/[resource]`
- Request Body: [필드 설명]
- Response: [필드 설명]

### 성능 요구사항
- 응답 시간: [목표값]
- 동시 사용자: [목표값]

### 의존성
- [외부 서비스/라이브러리]: [버전/용도]
```

### 릴리즈 계획 (Release Plan)
개발 일정과 마일스톤.

```markdown
## 릴리즈 계획
| 마일스톤 | 목표일 | 내용 |
|----------|--------|------|
| Alpha | [날짜] | [범위] |
| Beta | [날짜] | [범위] |
| GA | [날짜] | [범위] |
```

### 성공 지표 (KPI / Success Metrics)
기능 출시 후 성과를 측정할 지표.

```markdown
## 성공 지표
| 지표 | 현재값 | 목표값 | 측정 방법 |
|------|--------|--------|-----------|
| [지표1] | [현재] | [목표] | [방법] |
```

### 리스크 및 의존성 (Risks & Dependencies)
진행 중 발생할 수 있는 리스크와 외부 의존성.

```markdown
## 리스크 및 의존성
### 리스크
| 리스크 | 영향도 | 발생 확률 | 대응 방안 |
|--------|--------|-----------|-----------|
| [리스크1] | 상/중/하 | 상/중/하 | [대응] |

### 의존성
- [의존 항목]: [담당팀] — [상태]
```

### QA 기준 / 테스트 시나리오
QA팀이 참고할 테스트 기준.

```markdown
## QA 기준
### 테스트 시나리오
| ID | 시나리오 | 사전 조건 | 기대 결과 |
|----|----------|-----------|-----------|
| TC-01 | [시나리오] | [조건] | [결과] |

### 엣지 케이스
- [케이스 1]: [예상 동작]
```

### 시나리오 대분류 / 우선순위 매트릭스
시나리오가 많을 때 대분류와 우선순위를 정리하는 섹션.

```markdown
## 시나리오 대분류
| 대분류 | 설명 | 우선순위 |
|--------|------|----------|
| [카테고리1] | [설명] | P0 |
| [카테고리2] | [설명] | P1 |

우선순위: [카테고리1] > [카테고리2] > ...
```

### 에이전트/LLM Context 설계
AI 에이전트 기능일 경우, 에이전트에 제공할 Context를 정의하는 섹션.

```markdown
## 에이전트 Context 설계
### [프로세스명]에 제공할 Context
| Context | 설명 |
| --- | --- |
| [항목 1] | [어떤 데이터를 왜 제공하는지] |
| [항목 2] | [설명] |
```

### 상태 감지 / 완료 조건 매트릭스
각 액션의 완료를 시스템이 어떻게 감지하는지 정의.

```markdown
## 완료 감지 매트릭스
| 액션 | 완료 감지 방식 |
| --- | --- |
| [액션 1] | [감지 방법: 로그, 타이머, 사용자 입력 등] |
```

### 권한 및 접근 제어 (Permissions)
사용자 역할별 접근 권한.

```markdown
## 권한 및 접근 제어
| 기능 | Admin | Manager | User |
|------|-------|---------|------|
| [기능1] | RW | R | - |
```

### 경쟁사 분석 / 레퍼런스
참고한 경쟁 제품이나 레퍼런스.

```markdown
## 레퍼런스
| 제품/서비스 | 참고 포인트 | 차별점 |
|-------------|-------------|--------|
| [제품명] | [어떤 점을 참고] | [우리의 차이] |
```

## 섹션 추천 기준

PM에게 섹션을 추천할 때 아래 기준을 참고한다:

- **B2B SaaS 기능** → 권한/접근 제어, 기술 요구사항 추천
- **AI/에이전트 기능** → 에이전트 Context 설계, 상태 감지 매트릭스 추천
- **복잡한 플로우** → 시나리오 대분류/우선순위, 상태 감지 매트릭스 추천
- **사용자 대면 기능** → 성공 지표, 경쟁사 분석 추천
- **플랫폼/인프라** → 기술 요구사항 추천
- **MVP/빠른 출시** → Out of Scope, 릴리즈 계획 추천
- **대규모 프로젝트** → 리스크/의존성, QA 기준 추천
